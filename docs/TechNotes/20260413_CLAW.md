# 當大家還在學怎樣『養龍蝦』，模型公司正在學怎樣『不需要龍蝦』。

呢一波「龍蝦」（AI Agent），如果你只係喺度吵「有冇用」，其實已經慢咗半拍。
更準確嘅睇法，係佢正在經歷一個三波市場——
👉 第一波賣夢
👉 第二波賣門檻![](Gemini_Generated_Image_jfbw6ajfbw6ajfbw%20(2).png)
👉 第三波，正在形成緊
![](Gemini_Generated_Image_jfbw6ajfbw6ajfbw%20(1).png)

第一波：賣夢
早期敘事好乾淨：

AI 幫你做事
自動化營運
一人公司
被動收入
核心一句：
👉「唔識技術都用得到」
結果大家落場之後發現——要顧 prompt、要修流程、要 debug、要維護環境。
第一批人開始離場，於是出現反彈聲音：
👉「成日忙住養龍蝦，但龍蝦冇做嘢」
第二波：賣門檻
當「人人可用」撐唔住，敘事升級：

唔係工具唔得，係你唔識用
你冇用 AI 管 AI
你模型太弱
你冇 system thinking
關鍵詞變成：workflow / agent design / orchestration
呢一波睇落比較專業，亦都比較貼近現實。
但佢同時做咗一件事：
👉 將失敗責任，由產品轉移到使用者身上
第一波賣幻想，第二波賣能力。
到呢度，大部分討論會卡喺兩個極端——
「龍蝦係騙局」vs「只有高手先配用」
兩個結論都唔精準。
工程現實：呢樣嘢仲係 framework，唔係 product
如果你用過 OpenClaw 呢類系統，體感通常好一致：
👉 你唔係喺度用工具，你係喺度搭一個系統
你要處理嘅其實係 workflow orchestration、state/memory 管理、tool chaining、failure recovery。
呢啲問題嘅性質，比較接近分散式系統 + automation pipeline，而唔係一個即開即用嘅 AI 助手。
第三波：賣平台
接落嚟會見到嘅，唔再淨係教學或者課程，而係——

幫你包好 workflow 嘅平台
預設好嘅 agent 模板
可控成本嘅 execution layer
規範好嘅 tool ecosystem
簡單講：
👉 將「工程問題」產品化
典型特徵：
1️⃣ 將 orchestration 收埋
2️⃣ 限制自由度，換穩定性
3️⃣ 用平台規則取代個人能力
4️⃣ 開始收 subscription，唔再賣課
但要留意——第三波唔等於問題解決，佢只係將問題集中處理。
不過，第三波其實有兩種可能結局
將第三波理解成「平台收租」，係由商業模式角度睇。
但如果換成技術演進角度，會見到另一條線：
👉 第三波亦可能係「模型直接食咗 agent 呢一層」
理由好現實。
而家大部分 agent 喺度做嘅嘢，本質上係三件事——任務拆解、工具調用、流程串接。
而呢三件事，正在快速變成模型嘅內建能力。
一個直接訊號，係 Anthropic 四月發布嘅 Claude Mythos Preview。
呢個模型冇被包成 agent，淨係直接接上沙盒環境，就可以——
![](Gemini_Generated_Image_jfbw6ajfbw6ajfbw.png)
自己拆解漏洞分析任務
自己決定用咩工具
自己跑出概念驗證攻擊
OpenBSD 一個瞓咗 27 年嘅 TCP SACK 漏洞、FFmpeg 一個 16 年冇人發現嘅 H.264 漏洞、FreeBSD 一個 17 年舊嘅遠端執行漏洞，全部係咁樣被翻出嚟。
而呢啲嘢，過去通常需要——
👉 一整套 security agent 框架 + 人工調校 + 多層 prompt chain
換句話講：
當 reasoning 能力去到一個臨界點，好多睇落「要靠 agent 先做到」嘅嘢，會直接變成模型一個普通 prompt 就做到嘅嘢。
一個好現實嘅問題就出現咗：
👉 今日你優化咗半年嘅 agent workflow，下一個版本嘅模型可能直接吞咗佢
Agent 對邊個最有價值，答案唔係使用者
如果 agent 真係會被模型吸收，咁點解成個產業仲喺度大力推 agent？
一個比較少人講嘅角度：
👉 因為 agent 係模型公司收集真實世界操作數據嘅最佳渠道
包括——使用者實際點拆任務、工具實際點被串、邊啲流程會 fail、點解 fail。
呢啲資料對訓練下一代模型嚟講，比 agent 本身嘅營收更有價值。
所以三波結構，可以再讀一次：

第一波：賣夢（讓人進場）
第二波：賣門檻（篩選用戶）
第三波：收平台租 + 收操作數據（同時發生）
而第三波收完之後，下一步可能係——
👉 將呢一層直接收返入模型裡面
到嗰陣，而家大部分中間層嘅 agent 公司，面對嘅唔係競爭，而係消失。
回到最核心嘅判斷
AI Agent（龍蝦）係咪未來？
👉 係，幾乎可以肯定
但而家大部分人遇到嘅問題係：
👉 喺度用「未完成產品」，但用「成熟產品」嘅期待去評價佢
一個工程視角嘅判斷準則——
如果一個 AI 系統：成功率不可預測、成本不可控、需要大量人工介入
咁佢目前就仲停喺：
👉 framework / infra 層，而唔係 product 層
結論唔係悲觀，亦唔係樂觀，而係分層睇：
👉 技術喺度成長
👉 市場喺度收割
👉 平台喺度形成
👉 而模型，可能正準備將呢一切收返入自己內部
問題從來都唔係：
❌「龍蝦值唔值得做」
而係：
✅「你而家企喺邊一層」
